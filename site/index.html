mathom

Mathom listens, remembers, organizes, and helps you rediscover conversations, meetings, and ideas. It transcribes voice recordings with faster-whisper, summarizes them with Ollama, and shelves every recording in a searchable archive — all on your own hardware.

The idea

Everything worth keeping, shelved with care

In Tolkien's Shire, a mathom is anything you have no immediate use for but are unwilling to throw away — kept safe in the mathom. Yours go here: audio, transcript, summaries, tags, and follow-up chat, stored together as a single searchable keepsake.

Features

A real archive, not a junk drawer

Every capability runs on your own hardware. Bring a recording home and Mathom does the rest.

🎙️

Voice transcription

faster-whisper handles any common audio format, with FFmpeg under the hood. CPU by default, optional NVIDIA GPU acceleration.

✨

AI summaries

12 editable prompt templates — TL;DR, Meeting Minutes, Action Items, Email Draft, and more — powered locally by Ollama.

💬

Follow-up chat

Ask questions about any recording, grounded in its transcript. Every message is stored with its Mathom.

🔍

Full-text search

SQLite FTS5 across titles, transcripts, and summaries, with highlighted snippets so you find the right memory fast.

🏷️

Tags & collections

Favorites, collections, archive, and a timeline view keep your growing shelves tidy and browsable.

📤

Exports

Take any Mathom with you as Markdown, plain text, or JSON. Your mathom, your files.

👥

Optional SSO

Off by default. Enable Authentik single sign-on for private per-user archives with roles and MFA.

🐳

One Compose stack

A single Docker Compose stack — CPU by default, optional GPU overlay, and TrueNAS SCALE compatible.

How it works

From recording to remembered

Each upload becomes a Mathom and moves through four stages — entirely on your machine.

1

Upload

Drop in a voice recording. Audio is stored on a local volume.

2

Transcribe

faster-whisper turns speech into an accurate transcript.

3

Summarize

Ollama distills it using the prompt template you choose.

4

Shelve

It joins your searchable mathom, ready to rediscover.

Browser React UI → mathom nginx + FastAPI → Ollama (internal only) + SQLite + audio volume

Security & privacy

Local-first, by design

Mathom ships with safe defaults. The mathom container is the only exposed service, and Ollama is never published beyond the internal network.

  • ✓ Loopback-only by default. With auth off, the UI binds to 127.0.0.1 — reachable only from the host.
  • ✓ No telemetry, no cloud. The only outbound calls are backend → Ollama on the internal network.
  • ✓ Validated uploads. Checked by extension and content, size-capped, stored under server-generated names.
  • ✓ Rate limiting. Protects uploads, chat, search, and the login surface.
  • ✓ Optional Authentik SSO. Per-user archives, RBAC, and MFA when you want them.
Quick start

Bring your first recording home

Requirements: Docker with Compose v2. Optional: NVIDIA Container Toolkit for GPU acceleration.

# Clone the repository git clone https://github.com/MacNite/mathom.git cd mathom # Configure — defaults work out of the box cp .env.example .env make up # start the CPU stack make models # pull the Ollama model (first run only) # Open the app http://localhost:8080

GPU users: run make up-gpu instead.
